<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Verify the 2-pane shell, controls, and deep-link routing in a browser</name>
  <what-built>
    A desktop 2-pane master-detail shell (sidebar 35% default, clamped 25–50%; reading pane
    remainder) built on shadcn Resizable, with all filter controls relocated into the sidebar
    pinned header, an independently-scrolling article list, and a header-only reading-pane stub
    driven by hash-encoded naddr deep-link routing (selectedNaddr state + hashchange sync +
    reactive selectedArticle memo). Plans 01–03 implemented this; `npm run build` and
    `npx tsc --noEmit` pass.
  </what-built>
  <how-to-verify>
    Run `npm run dev` and open the local URL (respect the `/soveng/` base path). Then check
    each Phase 6 success criterion:

    1. LAYOUT-01/02/03 — Split + scroll: With 20+ articles loaded, confirm a 2-pane horizontal
       split (sidebar left, reading pane right) filling the full viewport. Scroll the sidebar
       list — the reading pane must NOT move, and vice versa (independent scroll). The page
       itself should not show an outer scrollbar. Drag the resize handle: the sidebar must
       stop at ~25% (min) and ~50% (max). The green-on-black terminal/CRT look (scanlines,
       flicker, monospace) must cover both panes.
    2. LAYOUT-04 — Controls in sidebar: Confirm the hashtag facets, the Match ALL/ANY (AND/OR)
       toggle, the ML on/off toggle, the confidence slider, the download-progress indicator,
       and the hidden-article count are all visible in the sidebar header and still filter the
       list when changed.
    3. LINK-01/03 — Deep link survives hard reload: Click an article — the URL gains a
       `#naddr1…` fragment and the reading pane shows that article's header (title + avatar +
       name + timestamp). Copy the URL, open a new tab (or hard-reload), and confirm the SAME
       article opens directly with NO 404 page under `/soveng/`.
    4. LINK-02 — Cold-load + back/forward: Hard-reload a deep-linked article URL and confirm
       the pane briefly shows `> resolving article from relays…` then auto-resolves to the
       article header as relays stream (no manual refresh). Edit the hash to a bogus naddr and
       confirm `[404] article not found on connected relays` appears once loading finishes.
       Select several articles, then use the browser Back and Forward buttons — the reading
       pane must follow the selection history.
  </how-to-verify>
  <resume-signal>Type "approved" if all four criteria pass, or describe any issues (which pane, which criterion) so a gap-closure plan can address them.</resume-signal>
</task>
