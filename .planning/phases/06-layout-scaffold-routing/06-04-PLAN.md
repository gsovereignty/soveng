<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Nostr relay → app (`article.image` tag) | Author-controlled URL string rendered as an `<img src>` in every visible row — untrusted input crossing into the user's browser network stack |
| Nostr relay → app (`profile.picture`) | Author-controlled avatar URL (already rendered via shadcn Avatar in prior phases; unchanged surface) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04-01 | Information Disclosure | SidebarRow `<img>` (author image) | mitigate | `referrerPolicy="no-referrer"` suppresses the Referer header; `loading="lazy"` prevents off-screen rows from issuing requests until scrolled into view (Pitfall 6) |
| T-06-04-02 | Tampering | SidebarRow image src (mixed content / non-https) | mitigate | `safeImageUrl` renders only `https://` URLs; http://, data:, relative, blob: are dropped with no request issued (Pitfall 7) |
| T-06-04-03 | Denial of Service | SidebarRow image (dead/404 URL → broken-icon clutter + layout shift) | mitigate | Fixed `width`/`height` reserve space (no CLS); `onError` hides the element so a broken URL never shows a broken-image icon |
| T-06-04-04 | Elevation of Privilege | Reading pane Markdown (XSS via untrusted body) | accept | Out of scope for this plan — ArticleBody (rehype-sanitize) is reused unchanged and not modified here; re-verified in the 06-07 human-verify plan |
</threat_model>
