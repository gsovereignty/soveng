<objective>
Add the mobile single-pane experience below the `md` breakpoint without disturbing the desktop 2-pane split.

Below `md`: the layout is a single full-screen pane showing the article list by default and swapping to a full-screen reading pane when an article is selected (MOBILE-01). The swap uses CSS visibility (`hidden`/`block` toggled by `md:` utilities + selection state) — both panes stay mounted so the list's scroll position survives a round-trip (MOBILE-02 / Pitfall 14). The mobile reader shows a terminal-styled `‹ back` control that returns to the list (MOBILE-02). The Resizable drag handle is suppressed below `md` (MOBILE-03 / Pitfall 5) — desktop keeps the exact 35/65 resizable split shipped in 06-02..05.

Purpose: Make the email-client layout usable on a 375px screen, where a side-by-side split and a 4px drag handle are unusable, while keeping the desktop layout byte-for-byte intact.
Output: A responsive shell in `App.tsx` and a mobile-only back control in `ReadingPaneStub`.
</objective>
