<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public Nostr relays → data layer (kind:0) | kind:0 `content` is author-controlled arbitrary JSON; `name`/`picture` fields are untrusted strings, possibly malformed, oversized, or hostile URIs (ASVS V5 input validation). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering / DoS | Malformed or non-object JSON in kind:0 content | mitigate | parseProfile wraps `JSON.parse` in try/catch, defaults to `{}`; all fields resolve to undefined fallbacks. Unit-tested with `"{not json"`. No throw can crash the reducer. |
| T-02-07 | DoS | Profile-event flood / many authors | mitigate | Author set is bounded by the article freeze (≤21 pubkeys); a SINGLE batched subscription (D-09) — not one-per-author — bounds WebSocket message volume. maxWait 5000 closes the sub. |
| T-02-08 | Tampering | Prototype-pollution-style keys in profile JSON | accept | Parsed object is read by explicit field access (`data.display_name`, `data.name`, `data.picture`) and cast `Record<string, unknown>`; no spread into a prototype-bearing target, no dynamic key assignment to app objects. Profile is a flat typed record. |
| T-02-09 | Information Disclosure | Hostile `picture` URI (`javascript:`, `data:`, `http:` on https site) | accept (Phase 2) / flag | Phase 2 stores `picture` verbatim and renders nothing. Phase 3 `<img src>` MUST guard: onError fallback + reject/upgrade non-https schemes. Flagged downstream. |
| T-02-10 | Spoofing | Profile event for a non-requested author | accept | SimplePool matchFilters drops events outside `authors:[...]`; signatures verified before onevent. Built-in. |

## Downstream (Phase 3) assumptions flagged
- `Profile.displayName` may be undefined — Phase 3 supplies the `npubEncode(pubkey).slice(0,12)+'...'` fallback (RESEARCH npub fallback example). Phase 2 deliberately leaves it undefined rather than baking display logic in.
- `Profile.picture` is unsanitized and may be a hostile/non-https URI — Phase 3 image rendering must guard.
</threat_model>
