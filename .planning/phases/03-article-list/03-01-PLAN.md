<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Nostr relay → app | Untrusted profile/article fields (title, summary, displayName, picture URL) cross into the browser; any author can write any value |
| npm registry → build | New `@radix-ui/react-avatar` dependency pulled by the shadcn CLI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering/Elevation (XSS) | ArticleCard title/summary/displayName render | mitigate | Render all untrusted strings as React text children only — never `dangerouslySetInnerHTML`. Acceptance criteria assert zero occurrences. |
| T-03-02 | Elevation (XSS via URL scheme) | AvatarImage `src` from untrusted `profile.picture` | mitigate | Value flows into an `<img src>` via Radix AvatarImage (no script execution); a `javascript:`/`data:` or broken/mixed-content URL fails the image load and AvatarFallback (monogram) renders. No URL is executed or fetched as code. |
| T-03-03 | Information Disclosure | Untrusted picture URL triggers outbound request (tracking pixel) | accept | Read-only static SPA, no credentials/cookies on cross-origin image loads; loading a public author avatar is the intended behavior. Low risk, no PII leaked. |
| T-03-04 | Denial of Service | Malformed pubkey breaks npubEncode | mitigate | `npubEncode` wrapped in try/catch with raw-hex slice fallback; card never throws. |
| T-03-SC | Tampering | `@radix-ui/react-avatar` install via shadcn CLI | mitigate | Blocking-human checkpoint (Task 2) verifies the package at npmjs.com/package/@radix-ui/react-avatar is the official @radix-ui scope package before acceptance. |
</threat_model>
