<objective>
Implement `useClassification(articles)` — the orchestration hook that ties the synchronous gates (Plan 01)
and the ONNX worker (Plan 02) together with a per-event-id result cache. It runs the cheap gates first
(franc language gate, then 500-word length gate), only dispatching survivors to the worker; caches every
result in a `useRef<Map>` keyed by event id (MLINF-03 — never re-infer on re-render); converts raw worker
scores to `'spam'` vs `'ham'` against a tunable threshold (default ~0.90, D-02/SPAM-02); and fails open on
every error path (SPAM-04). It also surfaces model-download progress and a model-failed flag for the UI plan.

Purpose: One hook owns the classification lifecycle and exposes a stable contract the App (Plan 05) and the
controls (Plan 04) consume — no classification state leaks into the Nostr reducer (ARCHITECTURE anti-pattern 1).
Output: `src/hooks/useClassification.ts` (new), `src/hooks/useClassification.test.ts` (new).

Interface contract (consumed by Plan 04 and Plan 05): `useClassification(articles: Article[])` returns
`{ map: Map<string, ClassificationLabel>; version: number; scores: Map<string, number>; downloadProgress: number | null; modelFailed: boolean }`.
The `version` counter increments on every map mutation so a `visibleArticles` memo depending on `version`
re-evaluates as results stream in. Spam/ham resolution against the threshold happens inside the hook when a
score arrives; re-thresholding without re-inference is achieved by storing raw `scores` and recomputing labels.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted article content → hook gates + postMessage | Article bodies feed franc, word-count, and the worker payload; the hook must bound work |
| Worker results → label decision | The hook trusts worker labels/scores to decide what the UI hides |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-PAYLOAD | Denial of Service | postMessage payload size | mitigate | Truncate text to content.slice(0,512) before postMessage (Performance trap: full-body transfer causes GC pressure); never transfer full article bodies |
| T-05-REINFER | Denial of Service | re-classification on re-render | mitigate | Cache by event id in useRef Map; skip ids already present so hashtag-filter toggles and threshold changes never re-run inference (MLINF-03) |
| T-05-FAILOPEN | Tampering | label decision on error | mitigate | 'error'/missing/'pending' fall through to shown via isHidden allowlist; a failed/absent worker cannot hide content (SPAM-04) |
| T-05-RDR | Spoofing | worker message handler | mitigate | Only act on messages with `type === 'result'` (and `type === 'progress'`) and a known id; ignore other message shapes |
</threat_model>
