<objective>
Establish the synchronous, dependency-light foundation for content filtering: the `ClassificationLabel`
type contract (consumed by every later plan), the `franc-min` language gate, and the 500-word length gate.
Both gates are pure, synchronous, and fully unit-testable — they run before any ONNX dispatch and never
touch a Worker. This plan implements LANG-01, LANG-02, and LEN-01 with hard fail-open semantics (D-03)
and the always-on length rule (D-05/D-06).

Purpose: Lock the label contract so the ONNX worker (Plan 02) and the hook (Plan 03) build against a fixed
type, and ship the cheap gates that keep non-English/short articles out of the expensive inference queue.
Output: `src/types/nostr.ts` (extended), `src/lib/languageDetect.ts` (new), `src/lib/languageDetect.test.ts` (new).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted article body → languageDetect | Any Nostr author can craft an arbitrarily large / adversarial Markdown body that this synchronous code parses with regexes on the main thread |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-RX | Denial of Service | stripNonNatural regexes in languageDetect.ts | mitigate | Use linear-time regexes only (no nested quantifiers); cap input to first ~20000 chars before regex/franc work so a hostile multi-MB body cannot freeze the main thread (ReDoS / unbounded work) |
| T-05-LBL | Tampering | ClassificationLabel hide allowlist | mitigate | isHidden is an explicit allowlist of exactly three hide labels; unknown/undefined labels fall through to shown (fail-open), so a malformed label can never silently hide content |
| T-05-SC | Tampering | npm install franc-min@6.2.0 | accept | franc-min is a well-known maintained package (wooorm), version pinned in STACK.md Package set; no [ASSUMED]/[SUS] entry in research |
</threat_model>
