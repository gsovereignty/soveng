# Requirements Archive — v1.1 Local ML Content Filtering

**Archived:** 2026-06-08 (milestone v1.1 shipped)
**Defined:** 2026-06-08
**Core Value:** Discover and read recent Nostr long-form articles, filtered by hashtag — with zero backend, served entirely as a static GitHub Pages site.

> This is a frozen snapshot of the v1.1 requirements as shipped. All requirements were
> satisfied. CTRL-06 was reinterpreted (folded into CTRL-05) rather than built as a
> separate control. See outcome notes below.

## v1.1 Requirements — All Complete

### Language Filtering

- [x] **LANG-01**: Non-English articles detected in-browser via franc-min and hidden from the list — *Validated (Phase 5, Plan 01)*
- [x] **LANG-02**: Detection strips Markdown/code, requires a minimum text length; undetermined or too-short articles fail open (treated as English, shown) — *Validated (Phase 5, Plan 01; MIN_DETECT_CHARS=200, 'und' = English)*

### Spam Filtering

- [x] **SPAM-01**: Articles classified spam/ham by an in-browser ML model (transformers.js ONNX); spam hidden — *Validated (Phase 5, Plans 02–03)*
- [x] **SPAM-02**: Conservative default confidence threshold (~0.90) so legitimate crypto/Nostr articles are not over-filtered — *Validated (Phase 5, Plan 06; GO verdict pinned SPAM_THRESHOLD=0.90)*
- [x] **SPAM-03**: Articles below a minimum text length bypass spam classification and are always shown — *Validated (Phase 5, Plan 03, gates-first)*
- [x] **SPAM-04**: Classification fails open — model-load failure, worker crash, or inference error leaves the article visible — *Validated (Phase 5, Plans 02–03)*

### Length Filtering

- [x] **LEN-01**: Articles shorter than 500 words hidden as an editorial long-form threshold; always on, independent of the content-filter toggle — *Validated (Phase 5, Plans 01 + 05)*

### ML Infrastructure

- [x] **MLINF-01**: Model runs off the main thread in a Web Worker so the UI never blocks or janks — *Validated (Phase 5, Plan 02, module-level worker singleton)*
- [x] **MLINF-02**: Model loads under the `/soveng/` base path (wasmPaths pinned, numThreads=1) and is cached across sessions — *Validated (Phase 5, Plans 02 + 06; live-URL smoke test, ORT 1.26.0-dev.20260416-b7804b056c)*
- [x] **MLINF-03**: Articles render immediately on fetch and hide progressively as results arrive; result cached per event id — *Validated (Phase 5, Plans 03 + 05)*

### Filter Controls & Transparency

- [x] **CTRL-01**: Content filtering is ON by default — *Validated (Phase 5, Plan 05, localStorage `!== 'false'` default-on)*
- [x] **CTRL-02**: Progress indicator shown while the model downloads on first visit — *Validated (Phase 5, Plan 04, shadcn Progress)*
- [x] **CTRL-03**: Reader displays a count of how many articles were filtered (hidden) — *Validated (Phase 5, Plans 04–05, hidden-count Badge)*
- [x] **CTRL-04**: User can toggle content filtering on/off (persists across reloads) — *Validated (Phase 5, Plan 05, localStorage-persisted Switch)*
- [x] **CTRL-05**: User can adjust the spam confidence threshold via a slider — *Validated (Phase 5, Plans 04–05, 0.50–0.99 slider, instant re-thresholding)*
- [x] **CTRL-06**: Reveal/recover false positives — *Validated (reinterpreted 2026-06-08). No separate "show hidden" control; the CTRL-05 spam-confidence slider at its maximum effectively disables spam filtering, surfacing any false positives. This is the recovery mechanism.*

**Phase 5** on 2026-06-08 (commit 700a576). v1.1 shipped as a single phase delivering the full
filtering feature plus controls.

## Deferred Beyond v1.1

### Spam Quality
- **SPAM-05**: Replace the SMS-trained spam model with a Nostr/long-form-trained classifier if false-positive rates remain high after launch feedback.
- **SPAM-06**: Per-article "why filtered" disclosure (signal + confidence).

### Filtering Beyond Content
- **MUTE-01**: Pubkey denylist / mute list to hide known-spam authors without ML.

### Carried from v1.0 Active (unrelated to ML filtering)
- **CONF-01**: User-configurable relay set
- **CONF-02**: Adjustable feed length
- **ENRICH-01**: Article summary/image on cards
- **ENRICH-02**: Clickable hashtag pills
- **ENRICH-03**: Per-relay connection status

## Out of Scope (v1.1)

| Feature | Reason |
|---------|--------|
| Server-side / build-time classification | Violates zero-backend, works-for-every-visitor constraint — all ML runs in the visitor's browser |
| Local Ollama / external LLM endpoint | Would only filter on machines running a model server; deployed site must filter for all visitors in-browser |
| Multi-language support (beyond English) | v1.1 scope is English-only; the language gate is a binary is-English test |
| Per-article spam-score badges on visible articles | Anti-feature — clutters the reading view; hidden-count + slider cover transparency |
| Re-classifying on every render / no result cache | Performance anti-pattern — classification cached per event id |
| Consent gate before model download | Filtering on by default; ambient progress indicator is sufficient disclosure |
| Training/fine-tuning a custom spam model this milestone | Deferred (SPAM-05) — ship best available browser model + high threshold first |
