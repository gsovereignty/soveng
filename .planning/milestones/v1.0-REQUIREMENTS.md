# Requirements Archive — v1.0 MVP

**Archived:** 2026-06-07 (milestone v1.0 shipped)
**Defined:** 2026-06-05
**Core Value:** Discover and read recent Nostr long-form articles, filtered by hashtag — with zero backend, served as a static GitHub Pages site.

> This is a frozen snapshot of the v1 requirements as shipped. All requirements were
> satisfied. See outcome notes for any adjustments made during execution.

## v1 Requirements — All Complete

### Data (Nostr fetching)

- [x] **DATA-01**: App connects to a fixed default set of relays (relay.damus.io, nos.lol, relay.nostr.band, relay.primal.net) over WebSocket — *Validated (Phase 2)*
- [x] **DATA-02**: App fetches kind:30023 events and presents the most recent — *Validated (Phase 2). Adjusted during milestone: quick task 260607-vqt lifted the fixed 21 cap and re-sorted by reply count.*
- [x] **DATA-03**: App dedupes addressable events by `kind:pubkey:d` coordinate — *Validated (Phase 2, first-arriving wins per D-03)*
- [x] **DATA-04**: App applies an EOSE/response timeout so an unresponsive relay never blocks rendering — *Validated (Phase 2, maxWait backstop timer)*
- [x] **DATA-05**: App resolves author display data by batch-fetching kind:0 profiles in a single subscription — *Validated (Phase 2, batched subscribeMany)*
- [x] **DATA-06**: App parses NIP-23 metadata with fallbacks for missing optional tags — *Validated (Phase 2, parseArticle fallbacks)*

### Article Display

- [x] **DISP-01**: Each article shows its title (with a sensible fallback when absent) — *Validated (Phase 3)*
- [x] **DISP-02**: Each article shows the author's name (display_name → name → truncated npub) and picture — *Validated (Phase 3)*
- [x] **DISP-03**: Each article shows a human-readable timestamp (`published_at` else `created_at`) — *Validated (Phase 3, formatTimestamp + 39 tests)*
- [x] **DISP-04**: Clicking an article expands its full body rendered as sanitized Markdown inline — *Validated (Phase 4, Accordion + ArticleBody)*
- [x] **DISP-05**: App shows distinct loading, empty-filter, and relay-error states — *Validated (Phases 2-4)*

### Hashtag Filtering

- [x] **FILT-01**: App derives a hashtag facet list from the `t` tags, lowercased/normalized — *Validated (Phase 4, buildFacets)*
- [x] **FILT-02**: Each hashtag facet shows a count of matching articles — *Validated (Phase 4, computeDynamicCounts)*
- [x] **FILT-03**: User can select/deselect hashtags via checkboxes to filter the list — *Validated (Phase 4, Checkbox facets)*
- [x] **FILT-04**: User can toggle between AND ("Match ALL") and OR ("Match ANY") combination — *Validated (Phase 4, ToggleGroup; AND predicate unit-tested after CR-01 fix)*

### Presentation & Delivery

- [x] **UI-01**: UI built with React + shadcn/ui, themed to a terminal aesthetic — *Validated (Phase 1, green-phosphor OKLCH + CRT chrome + JetBrains Mono)*
- [x] **DEPLOY-01**: App builds to static assets via Vite with a correct GitHub Pages `base` path — *Validated (Phase 1, base /soveng/)*
- [x] **DEPLOY-02**: GitHub Actions workflow builds and publishes to Pages, with SPA 404 fallback — *Validated (Phase 1)*

## Out of Scope (v1)

| Feature | Reason |
|---------|--------|
| Authoring / publishing articles | Read-only discovery client, not a writing tool |
| Login / signing / NIP-07 | No user identity needed to browse public content |
| Comments, likes, zaps, social interactions | Discovery only, not engagement |
| Pagination / infinite scroll beyond 21 | Fixed-size curated view |
| Server / backend / database | Static site only; all fetching client-side |
| External reader links (njump.me) | Reading happens inline, in-app |
