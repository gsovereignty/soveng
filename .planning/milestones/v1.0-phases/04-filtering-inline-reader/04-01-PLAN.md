| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering (XSS) | FilterBar tag labels (`#{tag}`) | mitigate | Tags rendered as React text children (JSX), never `dangerouslySetInnerHTML`; React escapes by default. Hashtags are already lowercased/normalized in the Phase 2 model |
| T-04-02 | Denial of Service | buildFacets/computeDynamicCounts over articles | accept | Result set capped at 21 articles by the data layer; facet/count loops are O(articles x tags) over a fixed tiny set — no practical DoS surface in a read-only client |
| T-04-03 | Information Disclosure | Local filter state | accept | No PII, no secrets, no persistence, zero backend — filter state is ephemeral UI-only |
| T-04-SC | Tampering | npm/shadcn installs (checkbox, toggle-group via `npx shadcn add`) | accept | No new npm packages added in this plan; shadcn add generates local component files using already-installed `radix-ui`.
