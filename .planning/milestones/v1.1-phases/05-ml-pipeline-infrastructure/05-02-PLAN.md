<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HF/jsDelivr CDN → Worker | ONNX model + WASM runtime fetched from third-party CDNs; availability and integrity are outside our control |
| main thread → Worker (postMessage) | App sends `{id, text}` derived from untrusted article bodies across the worker boundary |
| Worker → main thread (postMessage) | Worker returns labels/scores the App trusts to hide content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-WASM | Denial of Service | wasmPaths CDN load | mitigate | Pin to exact ORT version (Pitfall 3); fail-open catch posts label:'error' so a CDN outage or version mismatch shows all articles rather than blocking the app (SPAM-04) |
| T-05-NUMT | Denial of Service | ONNX threading on GitHub Pages | mitigate | numThreads=1 set before pipeline() — avoids SharedArrayBuffer requirement that GitHub Pages cannot satisfy (no COOP/COEP); prevents silent runtime failure |
| T-05-MSG | Tampering | worker onmessage handler | mitigate | Validate inbound message shape (string id + string text); ignore malformed messages instead of feeding arbitrary objects into the pipeline |
| T-05-KEYLOG | Information Disclosure | worker debug logging | mitigate | Log only event id + numeric score; never log article text/content (bodies may embed private keys from tutorial content) |
| T-05-SC | Tampering | npm install @huggingface/transformers@4.2.0 | accept | Canonical HF v3+ package, version pinned in STACK.md Package set; not [ASSUMED]/[SUS]; legacy @xenova/transformers explicitly rejected |
</threat_model>
