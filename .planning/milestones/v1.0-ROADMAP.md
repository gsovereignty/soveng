# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-06-07
**Phases:** 1-4
**Total Plans:** 9

## Overview

Four vertical MVP phases that each left the site deployable and incrementally more
capable: first a live Pages skeleton established the terminal theme and deploy pipeline;
then the data layer delivered clean normalized Nostr articles to the browser; then the
article list rendered real content end-to-end; finally hashtag faceting and inline
Markdown reading completed the full v1 experience.

Delivered: a zero-backend, terminal-styled Nostr long-form reader, live on GitHub Pages,
that streams kind:30023 articles from four public relays, resolves author profiles,
and lets readers filter by hashtag and read Markdown inline.

## Phases

### Phase 1: Scaffold & Deploy

**Goal**: A terminal-themed React + shadcn/ui app is live on GitHub Pages serving a placeholder page
**Mode:** mvp
**Depends on**: Nothing (first phase)
**Requirements**: UI-01, DEPLOY-01, DEPLOY-02
**Plans**: 2 plans

Plans:

- [x] 01-01: Scaffold Vite+React+shadcn, terminal theme (green-phosphor OKLCH, CRT chrome), reusable BootSequence placeholder
- [x] 01-02: Vite base path (/soveng/), SPA 404 fallback, GitHub Actions → Pages deploy

**Details:**
Established the green-phosphor OKLCH terminal palette via `@theme inline`, monospace-everywhere (JetBrains Mono self-hosted), full CRT chrome, and a live deployment at https://gsovereignty.github.io/soveng/ via GitHub Actions (upload-pages-artifact + deploy-pages, least-privilege permissions, SPA 404.html postbuild).

### Phase 2: Nostr Data Layer

**Goal**: The app connects to the default relay set, fetches and deduplicates kind:30023 articles, batch-resolves author profiles, and exposes a clean normalized article model with safe fallbacks
**Mode:** mvp
**Depends on**: Phase 1
**Requirements**: DATA-01, DATA-02, DATA-03, DATA-04, DATA-05, DATA-06
**Plans**: 3 plans

Plans:

- [x] 02-01: Article-streaming slice — deps + types + pool singleton + pure NIP-23 parse (tested) + reducer + context + useArticleFetch + App wiring (per-relay status, timer)
- [x] 02-02: Profile-resolution slice — parseProfile + PROFILE_RECEIVED (newest-wins) + batched single kind:0 useProfileFetch
- [x] 02-03: Refetch/retry slice — RESET correctness (Pitfall 3) + refetch() exposed + terminal-styled error/empty retry control

**Details:**
Module-level SimplePool singleton (StrictMode-safe), nostr-tools subpath imports only, freeze-then-dedup reducer, EOSE/maxWait backstop timer so an unresponsive relay never blocks render, batched single kind:0 subscription for all author pubkeys (newest-wins profile Map), and a RESET path that clears seenCoords to avoid stale-coordinate starvation on refetch.

### Phase 3: Article List

**Goal**: Real Nostr articles render end-to-end on the deployed Pages URL with title, author identity, timestamp, and correct loading/error/empty states
**Mode:** mvp
**Depends on**: Phase 2
**Requirements**: DISP-01, DISP-02, DISP-03, DISP-05
**Plans**: 2 plans

Plans:

- [x] 03-01: Card primitives slice — tested formatTimestamp (39 tests), shadcn Avatar, ArticleCard with title/author/timestamp fallbacks
- [x] 03-02: Integration slice — ArticleList streaming status line + arrival-order card mapping, boot-then-stream wiring, preserving error/empty/retry

**Details:**
Pure UTC ISO-8601 timestamp formatter (Intl.DateTimeFormat sv-SE singleton), shadcn Avatar with monogram fallback, npub-encode display fallback (try/catch hardened), boot-then-stream branch (BootSequence only while streaming && no articles), and profile upgrade-in-place as kind:0 events arrive. All untrusted content rendered as React text children — never dangerouslySetInnerHTML.

### Phase 4: Filtering & Inline Reader

**Goal**: Users can narrow the article list by hashtag with AND/OR logic and read any article's full Markdown content inline without leaving the page
**Mode:** mvp
**Depends on**: Phase 3
**Requirements**: FILT-01, FILT-02, FILT-03, FILT-04, DISP-04
**Plans**: 2 plans

Plans:

- [x] 04-01: Hashtag faceting slice — tested facets.ts (buildFacets count-rank + computeDynamicCounts) + sticky FilterBar (Checkbox tags, show-more, Match ANY/ALL ToggleGroup) + App.tsx filter state/useMemo chain + distinct empty-filter state
- [x] 04-02: Inline reader slice — react-markdown stack + shadcn Accordion + sanitized ArticleBody (no rehype-raw, links new-tab, full CRT) + controlled single-open Accordion with filter-clears-open

**Details:**
Count-ranked hashtag facets derived from the fetched articles' `t` tags, sticky top filter bar (a presentation delta from the original "sidebar" framing, D-01), AND/OR ToggleGroup defaulting to Match ANY, distinct empty-filter state (≠ relay-error), and an inline sanitized-Markdown reader using react-markdown + remark-gfm + rehype-sanitize (XSS-safe: no rehype-raw, `target=_blank` re-applied post-sanitization). Controlled single-open Accordion collapses an orphaned open article when the filter changes.

## Milestone Summary

**Decimal Phases:** None — all four phases executed as planned integers.

**Key Decisions:**

- nostr-tools SimplePool over NDK (bundle size: shiki/sandpack) and applesauce (RxJS overhead)
- Deploy-first ordering — Phase 1 establishes a live Pages URL so all later work is verifiable there
- Data layer isolated to Phase 2 before any UI consumes it — relay risks contained to one phase
- Module-level SimplePool singleton, never in React state (StrictMode-safe)
- nostr-tools subpath imports only (/pool, /kinds, /core, /nip19) — root barrel forbidden for tree-shaking
- Vite base `/soveng/` (project-subpath Pages model); GitHub Actions artifact deploy over gh-pages npm package
- Inline sanitized-Markdown reader over external njump.me link — keeps reading in-app
- Top filter bar over left/right sidebar (D-01 presentation delta — still satisfies FILT-01/02)
- rehype-sanitize (never rehype-raw) for untrusted article bodies

**Issues Resolved:**

- Pitfall-3 stale-coordinate starvation on refetch — RESET now clears seenCoords (regression-tested)
- StrictMode double-mount safety — pool singleton lives at module scope
- XSS surface on untrusted Markdown bodies — rehype-sanitize + post-sanitization link override

**Issues Deferred:**

- Per-relay connection status indicator (ENRICH-03, v2)
- User-configurable relay set (CONF-01, v2)
- Adjustable feed length (CONF-02, v2)
- Summary/image on cards (ENRICH-01) and clickable tag pills (ENRICH-02), v2

**Technical Debt Incurred:**

  during the milestone; the v1 requirement/constraint text ("21 most recent, newest first")
  still describes the original behavior. Reconcile requirement/constraint text in the next milestone.
